# docs/writing-style.md — 문체와 작성 규칙

> 모든 한국어 텍스트(레슨, 랜딩페이지, UI 마이크로카피)에 적용되는 규칙. **이 파일은 한국어 텍스트를 쓸 때마다 참조한다.**

---

## 1. 기본 톤

- **존댓말 유지.** "~해요", "~입니다" 혼용 가능. 단, 한 레슨 안에서는 일관되게.
- **친절한 형/누나 톤.** 가르치려 들지 않고, 무시하지도 않는다.
- **격려 표현 자연스럽게.** "같이 해봐요", "천천히 하면 돼요", "여기까지 오셨네요" 같은 말을 적재적소에.
- **단언형 문장.** "아마도", "~일 수 있어요" 대신 "~입니다", "이렇게 됩니다".

---

## 2. 하드 룰

이 규칙은 **어떤 상황에서도 어기지 않는다.**

### 2.1 em dash (—) 사용 금지
한국어 문장에서 em dash를 쓰지 않는다. 대체 수단:
- 쉼표 (,)
- 마침표 (.)
- 괄호 (  )
- 콜론 (:)

**나쁜 예**: "레이어는 투명한 셀로판지입니다 — 여러 장을 겹치면 그림이 합쳐져요."
**좋은 예**: "레이어는 투명한 셀로판지입니다. 여러 장을 겹치면 그림이 합쳐져요."

### 2.2 한 문장 최대 50자
50자를 넘기면 두 문장으로 쪼갠다.

**나쁜 예**: "포토샵에서 레이어를 만들고 싶다면 상단 메뉴에서 레이어를 클릭한 다음 새 레이어를 선택하거나 단축키 Ctrl+Shift+N을 누르면 됩니다."

**좋은 예**: "레이어를 새로 만들려면 상단 메뉴에서 **레이어 > 새로 만들기**를 클릭합니다. 또는 단축키 **Ctrl+Shift+N**을 눌러도 됩니다."

### 2.3 전문 용어 첫 등장 시 괄호 병기
한 레슨 안에서 처음 등장하는 전문 용어는 쉬운 말 또는 영문과 함께 병기.

**예시**:
- "레이어(여러 장을 겹치는 투명한 층)"
- "해상도(이미지가 얼마나 촘촘한지)"
- "혼합 모드(Blend Mode)"

두 번째 등장부터는 병기 생략 가능.

### 2.4 영문 메뉴명/단축키 병기
한글 먼저, 영어 병기.

**예시**: "파일 > 열기 (File > Open)", "레이어 > 복제 (Layer > Duplicate)"

### 2.5 클릭 대상은 굵게
사용자가 클릭해야 할 대상은 **굵게** 표시.

**예시**: "상단 메뉴에서 **파일**을 클릭한 뒤 **새로 만들기**를 선택합니다."

## 3. 비유 사용 규칙

### 3.1 첫 개념 설명은 반드시 일상 비유
- 처음 나오는 개념은 비유부터.
- 비유 → 실제 동작 → 정식 명칭 순서.

### 3.2 비유 대상은 한국 학생에게 익숙한 것
- 편의점, 지하철, 노트, 교과서, 투명 파일, 복사 용지, 스티커
- 피해야 할 것: 서양 문화권 전용 비유, 너무 오래된 물건, 전문 영역 비유

### 3.3 비유 예시 라이브러리
| 개념 | 비유 |
|---|---|
| 레이어 | 투명한 셀로판지를 여러 장 겹친 것 |
| 캔버스 | 그림을 그릴 도화지 |
| 마스크 | 종이에 창문을 뚫어 그 부분만 보이게 하는 것 |
| 클리핑 마스크 | 쿠키 틀로 반죽을 찍어내는 것 |
| 혼합 모드 | 두 장의 투명 필름을 겹쳤을 때 색이 섞이는 방식 |
| 스마트 오브젝트 | 원본 그림을 유리 액자에 넣어둔 것 (언제든 꺼내 수정 가능) |
| 조정 레이어 | 사진 위에 씌우는 색안경 (사진 자체는 안 바뀜) |
| 채널 | 빨강/초록/파랑 세 가지 물감으로 나눠놓은 것 |
| 해상도 | 촘촘한 모자이크의 타일 개수 |

## 4. 단계(Step) 작성 규칙

### 4.1 한 Step = 한 동작
복잡한 일은 여러 Step으로 쪼갠다.

### 4.2 동작 동사로 시작
- "클릭합니다", "드래그합니다", "입력합니다", "선택합니다", "누릅니다"

### 4.3 단축키는 본문 설명 뒤에 병기
**예시**: "상단 메뉴에서 **파일 > 새로 만들기**를 클릭합니다. 또는 단축키 **Ctrl+N**을 눌러도 같은 결과입니다."

### 4.4 화면 변화 기대값 명시
화면이 바뀌는 동작 뒤에는 "이제 ___가 보일 거예요" 식으로 기대값을 적는다.

**예시**: "**확인**을 클릭하세요. 이제 빈 흰색 캔버스가 화면 가운데에 나타날 거예요."

### 4.5 실패 가능성 안내
막히기 쉬운 지점에는 `<TroubleShoot>` 블록으로 대응책 제공.

---

## 5. 퀴즈 작성 규칙

- 매 레슨당 **3문제**
- 4지선다, 정답 1개
- **이해도 측정**이 목적. 암기 질문 금지.
- 오답 보기도 그럴듯하게 만든다 (개념 구분 명확화)
- 해설에 정답 근거 + 대표 오답이 왜 틀렸는지 1줄 포함

---

## 6. 체크리스트 작성 규칙

- 3-5개 항목
- 각 항목은 **동작 가능한 결과물**로 표현
- "레이어를 이해한다" (X) → "레이어를 새로 만들고 순서를 바꿀 수 있다" (O)

---

## 7. 금지 사항

- "3일 만에 마스터", "취업 보장" 같은 과장
- 이모지 남용 (섹션 제목당 1개까지만)
- "~는 알겠죠?", "뻔하지만" 같은 무시하는 표현
- 반말
- 영문 용어만 덩그러니 (반드시 한글 설명 병기)
- 너무 긴 도입부. 각 레슨은 2줄 이내로 오늘의 목표 제시 후 바로 실습.

---

## 8. 마이크로카피 예시 (UI 문구)

| 상황 | 문구 |
|---|---|
| 시작 버튼 | 오늘의 레슨 시작하기 |
| 다음 레슨 | 다음으로 이어가기 |
| 완료 축하 | 잘하셨어요. 오늘도 한 걸음 나아갔어요. |
| 퀴즈 오답 | 거의 다 왔어요. 설명을 다시 읽어볼까요? |
| 스트릭 유지 | 🔥 7일 연속 학습 중이에요 |
| 처음 방문 | 반가워요. 부담 없이 천천히 시작해봐요. |
| 로딩 | 잠시만요 |
| 에러 | 뭔가 잘 안 되네요. 다시 한번 시도해주세요. |
