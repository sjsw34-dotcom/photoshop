# docs/offline-learning-devices.md — 동영상 없이 작동하는 8가지 장치

> 이 사이트의 가장 중요한 설계 결정. 동영상이 없다는 "약점"을 8가지 장치로 상쇄한다. **이 파일은 레슨을 작성하거나 컴포넌트를 개선할 때, 학습 경험을 설계할 때 참조한다.**

---

## 왜 동영상 없이 설계하는가

### 동영상의 장점
- 시연이 직관적
- 보기 편함 (수동적 학습)

### 동영상의 단점 (우리 학습자에겐 치명적)
- 속도를 따라가기 힘듦. 자기 페이스가 아님
- 되돌려보려면 정확한 지점 찾기 어려움
- 스크린샷 한 장으로 끝날 내용을 2분 영상으로 보게 됨
- 검색 불가능. "이 단축키가 뭐였더라" 싶을 때 다시 찾기 힘듦
- 모바일에서 큰 데이터 소모
- 제작 비용이 매우 높음 (이 사이트는 한 사람이 집필)

### 그래서 "학습자 실습 중심"으로 설계
**모든 레슨은 학습자 본인 결과물 업로드로 마감**한다. 마스터가 찍은 스크린샷을 "잘 기억하는지"가 아니라 **학습자가 손으로 만든 결과를 갤러리에 쌓아가는 것**이 진짜 학습 증거다.

---

## 장치 1. 학습자 업로드 기반 실습 (가장 중요)

### 핵심
레슨 끝에 `<PracticeCompare>`가 있어, 학습자가 **두 장 모두 본인 사진**으로 올려 '시작 ⟷ 내 결과'를 좌우 드래그 슬라이더로 비교한다. **마스터는 이미지를 0장 제공한다.** 업로드 결과는 IndexedDB에 저장되어 대시보드 갤러리에 썸네일로 누적된다.

### 왜 강력한가
- 동영상을 "보는 것"보다 직접 만들어 올리는 것이 학습 효과가 크다.
- 본인 사진으로 작업하니 몰입도가 훨씬 높고, 진짜 내 포트폴리오가 된다.
- 마스터가 포토샵 전문가가 아니어도 커리큘럼을 돌릴 수 있다.
- 80 레슨 × 2장 = 160장의 개인 기록이 자동 생성된다.

### 구현
```mdx
<PracticeCompare
  lessonSlug="layer-mask-intro"
  goal="네 사진에서 배경만 마스크로 가려 올려보세요"
  startHint="배경이 단순한 인물/풍경 사진이 처음 연습에 좋아요"
  checklist={[
    "시작 사진을 준비했다",
    "배경이 자연스럽게 가려졌다",
    "원본 레이어는 그대로 남아 있다"
  ]}
/>
```

### 상태 4가지
- **pending**: 아무것도 안 올렸음 (기본)
- **deferred**: "지금 말고, 나중에" 눌러 대기 중
- **partial**: 한 장만 올라와 있음 → "한 장 더 올리면 슬라이더가 생겨요" 안내
- **uploaded**: 두 장 모두 완료 → 슬라이더 + 갤러리 완성 배지

### 동기부여 문구 (모든 레슨에 동일)
컴포넌트 상단에 고정된 배너:
> 🌱 이 한 장이 네 포트폴리오가 돼요. 네 사진으로 직접 만들어봐요. 완벽하지 않아도 괜찮아요. 올린 결과는 오직 너만 봐요. 오늘 한 장이 30일 뒤 30장이 돼요.

### 여건 (할 수 있게 하려면)
- 드래그·드롭 + 모바일 카메라 바로 열기 (`capture="environment"`)
- **"지금 말고, 나중에" 버튼 필수** (학습자 부담 0)
- 한 장만 올려도 저장됨 (partial 상태) — 부담 없이 진행
- 업로드 시 클라이언트에서 자동 리사이즈 (최대 1600px, JPEG 82%)
- 썸네일만 Zustand persist (localStorage), 원본은 IndexedDB `{slug}:before` / `{slug}:after` 키
- 갤러리에서 해당 카드 클릭 → 레슨으로 복귀해 다시 올리기

### frontmatter
모든 레슨은 `hasPractice: true`를 명시. (이 플래그로 대시보드 갤러리가 렌더링할 슬롯을 결정)

### 마스터 부담
이미지 준비: **0장**. MDX에 `goal` 한 문장과 `checklist` 3-4줄만 쓰면 완성.

## 장치 2. 마이크로 스크린샷 + 하이라이트 오버레이

### 핵심
**꼭 필요할 때만** 쓴다. 매 Step에 스크린샷 1장을 넣으려 하지 말 것. 화면 요소의 위치를 설명할 때, 그리고 학습자 업로드 전에 "출발점"을 보여줄 때만.

### 구현
- `<Screenshot>` 컴포넌트의 `highlight` prop 활용
- `highlight={[{ x: 15, y: 3, label: "1" }]}` 형태
- x, y는 0-100 (%). 반응형 대응

### 원칙
- 전체 창이 아닌 **필요한 영역만** 크롭한 스크린샷
- 여러 단계를 한 장에 몰아넣지 않기
- **"설명을 스크린샷으로 대신하려 하지 말 것."** 글로 충분하면 스크린샷은 빼라.

## 장치 3. 키보드 단축키 시각화

### 핵심
키보드 모양 그래픽으로 어떤 키를 누르는지 한눈에.

### 구현
- `<KeyboardShortcut keys={["Ctrl", "Shift", "N"]} />` 컴포넌트
- 각 키는 키캡 모양 박스로 렌더
- Windows/Mac 자동 감지로 Ctrl/Cmd 자동 전환
- 단축키가 나올 때마다 본문에 병기

### 배치
- 본문 중 단축키 언급 시 인라인
- 각 레슨 마지막에 "🔑 꼭 기억할 단축키" 섹션에 정리

---

## 장치 4. 연습 파일 번들

### 핵심
학습자가 업로드할 "내 결과"의 출발점이 되는 .psd / .jpg 파일을 제공. `<PracticeCompare>`의 `beforeSrc`는 보통 이 번들 안의 시작 이미지.

### 구성
```
/public/practice-files/{slug}/
├── start.psd      # 레슨 시작 시점 파일
├── start.jpg      # PracticeCompare의 beforeSrc로 사용
└── README.txt     # 간단한 사용 안내
```

### 배치
- `<PracticeFile>` 컴포넌트로 다운로드 버튼 (따라하기 섹션 상단)
- `<PracticeCompare beforeSrc="/practice-files/{slug}/start.jpg">` (실습 섹션)

### 초기 빌드에서 처리
실제 .psd 파일은 마스터가 직접 제작. 초기에는 placeholder .zip 파일 배치하고 파일 스펙을 README로 명시.

---

## 장치 5. Before/After 드래그 슬라이더

### 핵심
보정 전과 후를 드래그로 비교. 마스터가 제공하는 예시 비교엔 `<BeforeAfter>`, 학습자 자신의 결과 비교엔 `<PracticeCompare>` 내부에 같은 UI.

### 구현
- `<BeforeAfter beforeSrc="..." afterSrc="..." />` — 개념 설명용 (선택)
- `<PracticeCompare>` 내부의 슬라이더 — 업로드 후 자동 등장 (필수)

---

## 장치 6. 체크포인트 자가 점검

### 핵심
`<PracticeCompare>`의 `checklist` prop으로 "스스로 점검해보기" 항목 3-4개를 제시. 업로드 전/후 자신의 결과를 검증.

### 형태
```tsx
checklist={[
  "배경이 투명해졌다",
  "인물 가장자리가 깔끔하다",
  "원본 레이어를 건드리지 않았다"
]}
```

### 원칙
- 눈으로 확인 가능한 조건만
- 3-4개가 적당. 10개는 너무 많음
- 정답이 아니라 **자기 기준**을 세우는 연습

---

## 장치 7. 문제해결 박스 (TroubleShoot)

### 핵심
"이게 안 될 때" 접힌 박스로 막히는 지점 해결.

### 구현
- `<TroubleShoot>` 컴포넌트 (기본 접힘)
- 각 Step 내부 또는 레슨 말미 "😵 자주 묻는 문제" 섹션

### 수집 방법
- 초기에는 예상 가능한 문제를 마스터가 직접 작성
- 운영 후에는 학습자 피드백(👎 버튼)으로 수집해 추가

---

## 장치 8. AI 도우미 (Phase 6에서 구현)

### 핵심
각 레슨 우하단의 "🤖 물어보기" 버튼.

### 구현 (Phase 6)
- 클릭 시 챗 모달 오픈
- 현재 레슨의 제목, 섹션, 업로드한 실습 결과를 컨텍스트로 자동 포함
- Claude API로 질문 전송
- 학습자가 자신의 화면 스크린샷 업로드 가능 → "내 결과가 이래요, 맞나요?"

## 레슨 작성 시 필수 포함 체크리스트

모든 레슨을 작성할 때 다음을 포함했는지 자가 점검:

- [ ] **장치 1: `hasPractice: true` frontmatter + `<PracticeCompare>` 블록 (예외 없음)**
- [ ] 장치 2: 꼭 필요한 곳에만 스크린샷. 글로 대체 가능하면 빼기
- [ ] 장치 3: 본문에 단축키 언급 + 마지막에 단축키 섹션
- [ ] 장치 4: `practiceFile` frontmatter 설정 (없으면 명시적으로 null)
- [ ] 장치 5: 보정/합성 레슨이면 개념 설명용 `<BeforeAfter>` 추가 가능
- [ ] 장치 6: `<PracticeCompare>`의 `checklist` 3-4개
- [ ] 장치 7: "😵 자주 묻는 문제" 섹션에 2-4개 Q/A
- [ ] (장치 8은 Phase 6에서 자동 적용)

## 핵심 요약

**동영상 없이도 작동시키려면**, 레슨 한 개가 다음을 다 제공해야 한다:

> "출발점 이미지가 주어지고(연습 파일), 어떻게 만들지 단계별 지시가 있고(글+최소 스크린샷+단축키), 학습자가 직접 만들어 올려 비교하고(PracticeCompare), 스스로 기준으로 점검하고(체크리스트), 막혔을 때 해결책이 있는(TroubleShoot, FAQ) 자기 주도 학습 환경."

이게 영상보다 느릴 것 같지만, 실제로는 **자기 속도로 학습 + 본인 결과물 누적**이라는 이중 이점이 있다. 우리 학습자(23세, 기초 없음, 공백 2년)에게는 "내가 만든 증거가 쌓인다"는 피드백이 무엇보다 큰 동기부여다.
